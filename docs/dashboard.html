Дашборд и команды

Dashboard это экран управления агентом во время работы.

Он открывается после:

npm run dev

или:

npm run dev -- --profile=<slug>

Что видно на экране

Обычно dashboard выглядит примерно так:

Арина  22                                      userbot  
stage: Сходили один раз

interest       0
trust          0
attraction     0
annoyance     12
cringe         1


 12:00:01  привет                      
 12:00:10  ну привет                   


>

Верхняя строка

Арина  22                                      userbot

Значит:

  • Арина имя персонажа;
  • 22 возраст;
  • userbot режим Telegram;
  • `` агент работает;
  • `` агент на паузе.

Stage

stage: Сходили один раз

Это текущая стадия отношений.

Стадии:

met-irl-got-tg       встретились в реале, дала тг
tg-given-cold        дала тг, но холодная
tg-given-warming     начинает оттаивать
convinced            согласилась встретиться
first-date-done      сходили один раз
dating-early         только начали встречаться
dating-stable        стабильная пара
long-term            давно вместе
dumped               отшила / разрыв

Счётчики отношений

interest

Интерес.

Если высокий ей любопытнее отвечать.

trust

Доверие.

Если высокий меньше обороны и меньше холодных реакций.

attraction

Притяжение.

Влияет на тепло, флирт, инициативу.

annoyance

Раздражение.

Растёт от давления, спама, нытья, токсичности.

cringe

Кринж.

Растёт от странных, неуместных, слишком навязчивых сообщений.

Лента событий

В центре dashboard показывает последние события:

 текст

Входящее сообщение.

 текст

Исходящее сообщение агента.

 игнор (...): текст

Агент решил не отвечать.

i текст

Информационное событие.

! текст

Ошибка.

Как писать команды

Внизу есть строка:

>

Пиши туда команды. Все команды начинаются с :.

Пример:

:status

Нажми Enter.

Главные команды

:help

Показать список команд.

:help

:status

Показать текущее состояние.

:status

Полезно, если агент молчит и ты не понимаешь почему.

:why [chatId]

Показать объяснение текущего состояния/причин поведения.

:why

Можно указать chatId:

:why 123456789

Используй когда:

  • она игнорирует;
  • отвечает слишком холодно;
  • долго не читает;
  • непонятно, спит она или занята.

:debug [chatId]

Показать отладочную информацию.

:debug

Обычно там полезны:

  • история;
  • presence;
  • daily-life;
  • conflict;
  • realism context.

:pause

Поставить агента на паузу.

:pause

После этого он перестаёт отвечать.

:resume

Снять паузу.

:resume

:reset

Сбросить отношения и конфликт.

:reset

Обычно используется если:

  • всё сломалось в диалоге;
  • слишком высокий annoyance;
  • слишком высокий cringe;
  • персона ушла в dumped.

Важно: reset чистит часть состояния. Используй осознанно.

:stage <id>

Сменить стадию отношений.

:stage first-date-done

Примеры:

:stage tg-given-cold
:stage tg-given-warming
:stage dating-early

:wake [chatId]

Разбудить агента ночью.

:wake

Обычно ночью персона спит. :wake временно разрешает ей проснуться.

:amnesia <мин> [chatId]

Забыть последние N минут.

:amnesia 15

Используется если:

  • она отправила лишнее;
  • ты тестировал странные сообщения;
  • нужно откатить короткий кусок диалога.

:cringe

Показать уровень кринжа.

:cringe

:relationship

Показать текущую стадию и score.

:relationship

:persona

Показать начало persona.md.

:persona

:log

Показать последние строки сегодняшнего лога.

:log

Логи лежат здесь:

data/<slug>/log/YYYY-MM-DD.md

Userbot-команды

Эти команды работают только в userbot mode.

В bot mode они недоступны или ничего не сделают.

:read [chatId]

Пометить сообщения прочитанными.

:read

Если указан chatId:

:read 123456789

Это полезно для left-on-read поведения.

:block [chatId]

Заблокировать контакт.

:block

:unblock [chatId]

Разблокировать контакт.

:unblock

:clear-chat [chatId] [--revoke]

Очистить историю чата.

:clear-chat

Удалить для обоих:

:clear-chat --revoke

Осторожно: это разрушительное действие.

:report-spam [chatId]

Пожаловаться на спам.

:report-spam

:delete-last [chatId] [--local]

Удалить последнее сообщение агента.

:delete-last

Только локально:

:delete-last --local

:edit-last <текст>

Редактировать последнее сообщение агента.

:edit-last не, я потом отвечу

:sticker [chatId]

Отправить случайный стикер из библиотеки.

:sticker

Библиотека:

data/<slug>/stickers/library.md

:quit / :exit

Остановить агента и выйти.

:quit

или:

:exit

Что делать если агент молчит

Порядок проверки:

  1. Напиши :status.
  2. Напиши :why.
  3. Проверь, не стоит ли пауза.
  4. Проверь, не спит ли она.
  5. Проверь annoyance и cringe.
  6. Проверь :log.
  7. Если userbot попробуй :read.
  8. Если всё плохо :reset.

Что делать если она отвечает слишком много

Проверь:

  1. vibe в config.json.
  2. Stage возможно слишком тёплая стадия.
  3. speech.md добавь, что она отвечает коротко.
  4. communication.md добавь, что не поддерживает спам.

Сейчас runtime умеет message coalescing: если пользователь прислал много сообщений подряд, старый запланированный ответ отменяется и агент отвечает на пачку.

Что делать если она слишком холодная

  1. Смотри annoyance и cringe.
  2. Смотри stage.
  3. Попробуй :stage tg-given-warming.
  4. Если нужно полностью обнулить :reset.
  5. В config.json можно поставить vibe: "warm".

Горячие клавиши

  • Enter отправить команду.
  • Ctrl+C выйти.

Следующий шаг

Прочитай realism.md, чтобы понять, почему агент не просто отвечает, а принимает решение.
