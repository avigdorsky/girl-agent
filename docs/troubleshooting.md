# Troubleshooting

Эта страница для ситуаций, когда что-то не работает.

Иди по шагам. Не пытайся чинить всё сразу.

## Быстрая диагностика

Сначала проверь базовые вещи.

## 1. Проект запускается?

```powershell
npm run dev
```

Если ошибка сразу в терминале  смотри текст ошибки.

## 2. TypeScript проходит?

```powershell
npm run typecheck
```

Если тут ошибка, сначала чинится код/типы.

## 3. Есть профиль?

```powershell
npm run dev -- --list
```

Если профилей нет  запусти wizard:

```powershell
npm run dev
```

## 4. Правильный профиль запускаешь?

```powershell
npm run dev -- --profile=arina
```

## 5. Есть ошибки в dashboard?

Ошибки отображаются строками с `!`.

Также смотри:

```text
data/<slug>/log/YYYY-MM-DD.md
```

## Агент не отвечает

Возможные причины:

1. Он на паузе.
2. Персона спит.
3. Персона занята.
4. Слишком высокий `annoyance` или `cringe`.
5. Stage слишком холодная.
6. Активен conflict/cold period.
7. LLM API не отвечает.
8. Telegram adapter не получил сообщение.
9. Ты пишешь не в private chat.

Что делать:

1. В dashboard напиши:

```text
:status
```

2. Потом:

```text
:why
```

3. Если пауза:

```text
:resume
```

4. Если она спит:

```text
:wake
```

5. Если отношения сломались:

```text
:reset
```

6. Если userbot и нужно просто прочитать:

```text
:read
```

## Агент отвечает слишком много

Причины:

- `vibe` стоит `warm`;
- stage слишком тёплая;
- `speech.md` разрешает длинные ответы;
- модель сама разбивает ответ на много bubbles;
- активный диалог снижает задержки.

Что делать:

1. В `config.json` поставь:

```json
"vibe": "short"
```

2. В `speech.md` добавь:

```markdown
Отвечает коротко, обычно одним сообщением. Не пишет длинные объяснения.
```

3. В `communication.md` добавь:

```markdown
Если человек закидывает много сообщений подряд, не отвечает на каждое отдельно.
```

4. Проверь, что runtime обновлён: должен работать message coalescing.

## Агент отвечает на каждое сообщение из пачки

Так быть не должно.

Нормальное поведение:

```text
он: ну
он: давай
он: ответь
она: ты чего
```

Проверь:

1. Что запущена новая версия кода.
2. Что `npm run typecheck` проходит.
3. Что ты перезапустил `npm run dev` после обновления.
4. Что нет старого процесса в другом терминале.

На Windows иногда старый процесс остаётся открытым в другом окне. Закрой лишние терминалы.

## Галочки прочитано не появляются

Это касается userbot mode.

Проверь:

1. Ты точно в `userbot` mode?
2. В `config.json` есть `sessionString`?
3. Нет ошибок в dashboard?
4. Сообщение пришло в private chat?
5. Команда работает вручную?

Попробуй:

```text
:read
```

Если после `:read` галочки появились  readHistory работает, проблема в логике поведения.

Если не появились:

- возможно peer не резолвится;
- возможно Telegram не применил readHistory;
- возможно это не тот чат;
- возможно аккаунт/сессия нестабильна.

## Реакции не ставятся

Причины:

- чат не поддерживает реакции;
- реакция запрещена Telegram;
- peer не резолвится;
- это bot mode и Telegram API не разрешил reaction;
- ошибка проглочена адаптером как безопасный no-op.

Проверь dashboard. Если критической ошибки нет, значит реакция просто не поддержалась в этом контексте.

## Typing не видно

Typing может быть слишком коротким или Telegram может не показать его.

Проверь:

- агент реально отвечает с задержкой;
- `tick.typing=true`;
- Telegram клиент открыт в момент ответа;
- это private chat.

## Bot не запускается

Ошибка может быть:

```text
BOT_TOKEN missing
```

Что делать:

1. Открой `data/<slug>/config.json`.
2. Проверь:

```json
"mode": "bot",
"telegram": {
  "botToken": "..."
}
```

3. Если токена нет  создай нового бота через `@BotFather`.
4. Вставь токен.
5. Перезапусти:

```powershell
npm run dev -- --profile=<slug>
```

## Userbot не логинится

Частые причины:

- неправильный `api_id`;
- неправильный `api_hash`;
- телефон не в международном формате;
- код Telegram устарел;
- неправильный 2FA пароль;
- Telegram временно ограничил входы.

Что делать:

1. Проверь `api_id` и `api_hash` на `my.telegram.org`.
2. Телефон вводи так:

```text
+79991234567
```

3. Запусти wizard заново.
4. Вводи свежий код.
5. Если много раз ошибся  подожди.

## LLM API ошибка

Примеры:

```text
API key invalid
quota exceeded
model not found
rate limit
```

Что делать:

1. Проверь ключ в `config.json`.
2. Проверь баланс у провайдера.
3. Проверь название модели.
4. Попробуй другую модель.
5. Попробуй другой provider.

## Модель отвечает как ChatGPT

Причины:

- слабый `persona.md`;
- пустой `speech.md`;
- слишком тёплый `vibe`;
- модель плохо следует стилю;
- в prompt мало запретов.

Что делать:

1. Открой `speech.md`.
2. Добавь конкретные правила.
3. Открой `persona.md`.
4. Добавь характер, вкусы, раздражители.
5. Открой `communication.md`.
6. Добавь границы.

Пример:

```markdown
Не пишет "конечно", "я понимаю", "рада помочь".
Не объясняет свои чувства длинно.
Если тема скучная, отвечает сухо или игнорирует.
```

## Агент слишком часто игнорирует

Проверь:

```text
:status
```

Если высокий `annoyance` или `cringe`, это причина.

Решения:

1. Снизить конфликт:

```text
:reset
```

2. Сменить стадию:

```text
:stage tg-given-warming
```

3. Поставить `vibe`:

```json
"vibe": "warm"
```

4. В `communication.md` ослабить правила игнора.

## Медиа не работает

Фото/voice/sticker/video приходят как media event.

Проверь:

1. Это поддерживаемый тип?
2. В bot mode Telegram дал fileId?
3. В userbot mode файл доступен для скачивания?
4. Модель поддерживает vision, если нужно описание фото?
5. Нет ли ошибки в dashboard?

## Стикеры не отправляются

Проверь:

```text
data/<slug>/stickers/library.md
```

Если библиотека пустая, отправлять нечего.

Попробуй прислать стикер персонажу в primary chat, чтобы он попал в библиотеку.

## MCP не работает

Проверь:

1. Есть ли MCP в `config.json`.
2. Есть ли API key.
3. Правильный ли id инструмента.
4. Не падает ли MCP при старте.

Пример:

```json
"mcp": [
  {
    "id": "exa",
    "secrets": {
      "EXA_API_KEY": "..."
    }
  }
]
```

## Как полностью сбросить профиль

Осторожно: это удалит персонажа.

PowerShell:

```powershell
Remove-Item -Recurse -Force data\<slug>
```

Потом:

```powershell
npm run dev
```

## Как мягко сбросить отношения

В dashboard:

```text
:reset
```

Это лучше, чем удалять профиль.

## Как поменять LLM provider

Открой:

```text
data/<slug>/config.json
```

Измени:

```json
"llm": {
  "presetId": "openai",
  "proto": "openai",
  "baseURL": "",
  "apiKey": "...",
  "model": "..."
}
```

Потом перезапусти.

## Как поменять bot на userbot

Самый безопасный путь:

1. Создай новый профиль через wizard.
2. Выбери userbot.
3. Перенеси вручную старые файлы:

```text
persona.md
speech.md
communication.md
memory/long-term.md
```

Можно редактировать `config.json` руками, но новичкам лучше не надо.

## Когда создавать issue

Создавай issue, если:

- `npm run typecheck` падает на чистом проекте;
- wizard ломается на одном и том же шаге;
- Telegram login не работает при правильных данных;
- read receipts не работают даже через `:read`;
- runtime падает с ошибкой.

В issue приложи:

- версию Node.js;
- версию npm;
- режим `bot` или `userbot`;
- текст ошибки;
- кусок dashboard log;
- `config.json` без секретов.
